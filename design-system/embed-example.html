📋 Como Incorporar o Formulário

Este documento mostra as diferentes formas de incorporar o formulário forms-embed.html em outro site.

🎯 Método 1: iFrame (Recomendado para Segurança)

✅ Melhor escolha para: Incorporar o formulário em sites de terceiros mantendo isolamento total e segurança.

Vantagens:

  • Isolamento total de estilos CSS
  • Segurança (sandbox protection)
  • Fácil de implementar
  • Sem conflitos com o site hospedeiro

Desvantagens:

  • Altura fixa (pode não se ajustar automaticamente)
  • Limitações de localStorage/sessionStorage em alguns navegadores
  • Pode ser bloqueado por Content Security Policy

Código de Incorporação:

<!-- Incorporar via iFrame -->
<iframe
    src="https://seudominio.com/forms-embed.html"
    width="100%"
    height="1400px"
    frameborder="0"
    allow="clipboard-write"
    sandbox="allow-scripts allow-forms allow-same-origin allow-popups"
    style="border: none; max-width: 800px; margin: 0 auto; display: block;"
    title="Formulário de Qualificação"
></iframe>

Demonstração (iFrame):

💡 Dica: Ajuste o atributo height conforme necessário para evitar scroll duplo.

🔧 Método 2: Embed Direto via JavaScript

⚠️ Atenção: Este método pode causar conflitos de CSS com o site hospedeiro. Use apenas se você controlar ambos os domínios.

Vantagens:

  • Integração mais fluida com o site
  • Altura dinâmica automática
  • Sem limitações de localStorage

Desvantagens:

  • Possibilidade de conflitos CSS
  • Scripts externos podem afetar o site
  • Menor segurança

Código de Incorporação:

<!-- Container onde o formulário será carregado -->
<div id="flux-aurora-form-container">
    <p style="text-align: center; padding: 2rem;">Carregando formulário...</p>
</div>

<!-- Script de incorporação -->
<script>
(function() {
    const FORM_URL = 'https://seudominio.com/forms-embed.html';

    fetch(FORM_URL)
        .then(response => {
            if (!response.ok) {
                throw new Error('Erro ao carregar formulário');
            }
            return response.text();
        })
        .then(html => {
            // Criar um documento temporário para parsear o HTML
            const parser = new DOMParser();
            const doc = parser.parseFromString(html, 'text/html');

            // Extrair apenas o conteúdo do body
            const bodyContent = doc.body.innerHTML;

            // Inserir no container
            const container = document.getElementById('flux-aurora-form-container');
            container.innerHTML = bodyContent;

            // Executar scripts do formulário
            const scripts = container.querySelectorAll('script');
            scripts.forEach(script => {
                const newScript = document.createElement('script');

                if (script.src) {
                    newScript.src = script.src;
                } else {
                    newScript.textContent = script.textContent;
                }

                // Adicionar ao final do body para garantir execução
                document.body.appendChild(newScript);
            });

            // Copiar estilos do head
            const styles = doc.querySelectorAll('style, link[rel="stylesheet"]');
            styles.forEach(style => {
                const newStyle = style.cloneNode(true);
                document.head.appendChild(newStyle);
            });

            console.log('✅ Formulário carregado com sucesso');
        })
        .catch(error => {
            console.error('❌ Erro ao carregar formulário:', error);

            const container = document.getElementById('flux-aurora-form-container');
            container.innerHTML = `
                <div style="text-align: center; padding: 2rem; background: #f5e8e8; border-radius: 8px;">
                    <p style="color: #5c3333; font-weight: 600; margin-bottom: 0.5rem;">
                        Erro ao carregar formulário
                    </p>
                    <p style="color: #6b6b6b; font-size: 0.875rem;">
                        Tente recarregar a página ou entre em contato conosco.
                    </p>
                </div>
            `;
        });
})();
</script>
⚠️ CORS: Para este método funcionar, o servidor que hospeda forms-embed.html precisa ter headers CORS configurados: 
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type

🌐 Método 3: Embed via Script Tag (Widget)

💡 Melhor para: Distribuir o formulário como um widget reutilizável em múltiplos sites.

Neste método, você cria um script centralizado que pode ser incorporado com uma única linha de código.

1. Criar arquivo embed-widget.js no seu servidor:

(function() {
    'use strict';

    // Configuração
    const FORM_URL = 'https://seudominio.com/forms-embed.html';
    const CONTAINER_ID = 'flux-aurora-form';

    // Criar container se não existir
    let container = document.getElementById(CONTAINER_ID);
    if (!container) {
        container = document.createElement('div');
        container.id = CONTAINER_ID;
        document.currentScript.parentNode.insertBefore(container, document.currentScript);
    }

    // Criar iframe
    const iframe = document.createElement('iframe');
    iframe.src = FORM_URL;
    iframe.width = '100%';
    iframe.height = '1400px';
    iframe.frameBorder = '0';
    iframe.setAttribute('allow', 'clipboard-write');
    iframe.setAttribute('sandbox', 'allow-scripts allow-forms allow-same-origin allow-popups');
    iframe.style.cssText = 'border: none; max-width: 800px; margin: 0 auto; display: block;';
    iframe.title = 'Formulário de Qualificação';

    // Inserir iframe no container
    container.innerHTML = '';
    container.appendChild(iframe);

    console.log('✅ Flux Aurora Form Widget carregado');
})();

2. Incorporar no site de destino:

<!-- Basta adicionar uma linha de código -->
<script src="https://seudominio.com/embed-widget.js" async></script>

3. (Opcional) Especificar onde o formulário será inserido:

<!-- O widget será inserido dentro desta div -->
<div id="flux-aurora-form"></div>

<script src="https://seudominio.com/embed-widget.js" async></script>

🔒 Configuração de Segurança no Servidor N8N

Para o formulário funcionar em embed, configure os headers CORS no webhook N8N:

Opção 1: Configurar no N8N Workflow

Adicione um nó "Set Headers" antes da resposta do webhook:

{
  "Access-Control-Allow-Origin": "*",
  "Access-Control-Allow-Methods": "POST, OPTIONS",
  "Access-Control-Allow-Headers": "Content-Type, Accept"
}

Opção 2: Configurar no Nginx (se aplicável)

location /webhook/ {
    # Headers CORS
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods 'POST, OPTIONS';
    add_header Access-Control-Allow-Headers 'Content-Type, Accept';

    # Responder OPTIONS com 204
    if ($request_method = 'OPTIONS') {
        return 204;
    }

    # Proxy para N8N
    proxy_pass http://localhost:5678;
}
⚠️ Produção: Em produção, substitua * por domínios específicos: 
Access-Control-Allow-Origin: https://seusite.com, https://outro-site.com

✅ Checklist de Implementação

  1. Hospedar o formulário:
    • Upload forms-embed.html para seu servidor
    • Verificar que o arquivo é acessível via HTTPS
    • Testar abertura direta no navegador
  2. Configurar CORS no webhook N8N:
    • Adicionar headers CORS no workflow
    • Testar requisição POST via Postman/Insomnia
  3. Escolher método de incorporação:
    • iFrame para máxima segurança
    • JavaScript direto para integração fluida
    • Widget para distribuição em múltiplos sites
  4. Testar em diferentes navegadores:
    • Chrome/Edge
    • Firefox
    • Safari
    • Mobile (iOS Safari, Chrome Android)
  5. Verificar funcionalidades:
    • Validação de campos funciona
    • Máscara de telefone aplica corretamente
    • Dropdowns abrem e fecham sem problemas
    • Envio do formulário completa com sucesso
    • Redirecionamento após envio funciona

🐛 Troubleshooting

Problema: Formulário não carrega no iframe

Causa: Content Security Policy (CSP) bloqueando

Solução: Adicionar header no servidor que hospeda forms-embed.html:

Content-Security-Policy: frame-ancestors 'self' https://dominio-permitido.com

Problema: Erro CORS ao enviar formulário

Causa: Webhook N8N não configurado para CORS

Solução: Seguir seção "Configuração de Segurança no Servidor N8N" acima

Problema: sessionStorage não funciona em iframe

Causa: Navegador bloqueia storage em contexto de terceiros

Solução:

  1. Passar dados via URL query params
  2. Usar postMessage para comunicação entre janelas
  3. Ou hospedar formulário no mesmo domínio

Problema: Dropdowns não clicáveis no iframe

Causa: z-index muito alto causando conflitos

Solução: Já corrigido em forms-embed.html (z-index reduzido de 10000 para 1000)

Problema: Estilos quebrados em embed direto

Causa: Conflito de CSS com site hospedeiro

Solução:

  1. Usar iFrame ao invés de embed direto
  2. Ou adicionar prefixo namespace em todas as classes CSS
  3. Ou usar Shadow DOM (técnica avançada)
